What is LaneLayer?

LaneLayer is a Bitcoin-anchored execution environment that enables you to build and deploy applications (called "lanes") that process transactions independently while being anchored to Bitcoin's security and decentralization.

Think of it as a way to build decentralized applications that leverage Bitcoin's security model while providing the flexibility of smart contract-like execution.

What is a Derived Lane?

A derived lane is a containerized application that runs independently but receives transaction data from the Core Lane (the main Bitcoin-anchored execution layer).

Key Characteristics:
  • Runs as a separate HTTP server container
  • Receives transaction data via the /submit endpoint
  • Processes data independently (like this NFT minting app)
  • Can query Core Lane state when needed
  • Deployed using the LaneLayer CLI

Architecture Flow:

Bitcoin ‚Üí Core Lane ‚Üí Derived Lane (this app) ‚Üí Your Logic

How This NFT App Works

1. Minting NFTs

You can mint NFTs in two ways:

  • Direct API: Use the web interface or POST to /mint endpoint
  • Via Core Lane: Send transaction data to Core Lane, which forwards it to this app via /submit

2. Storage

NFTs are currently stored in-memory (for simplicity). In production, you'd use a database or persistent storage.

3. API Endpoints

  • GET /health - Health check
  • POST /mint - Mint a new NFT
  • GET /nfts - List all NFTs
  • GET /nft/{token_id} - Get specific NFT
  • POST /submit - Receive transactions from Core Lane

Via Core Lane (When Configured)

When Core Lane is configured to send transactions to this derived lane, they are forwarded to the /submit endpoint. The endpoint is ready to receive and process submissions in the format that Core Lane sends. For now, you can test this using the lane submit command or by sending POST requests directly to /submit.

Note: The full Core Lane ‚Üí Runner ‚Üí Container integration requires additional configuration. The /submit endpoint is implemented and ready, but Core Lane needs to be configured to route transactions to this specific container.

Getting Started with LaneLayer

To build your own derived lane:

  1. Install the CLI: npm install -g @lanelayer/cli
  2. Create a project: lane create myapp --template python
  3. Develop your app in app.py
  4. Test locally: lane up dev
  5. Build for production: lane build prod
  6. Deploy: lane push <registry>

Check out the LaneLayer CLI documentation for more details.
